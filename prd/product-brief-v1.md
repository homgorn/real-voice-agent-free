# VoiceAgent Product Brief v1

Дата: 9 марта 2026

## 1. Продуктовая идея

`VoiceAgent` — open-core платформа голосовой автоматизации для SMB-сервисного бизнеса.

Она должна решать не абстрактную задачу "сделать голосовой AI", а конкретные бизнес-задачи:

- ответ на пропущенные и входящие звонки
- запись на услугу
- квалификация лида
- ответы на частые вопросы
- перевод на человека при необходимости
- напоминания и подтверждение записи
- итог разговора и синхронизация с CRM/календарём

## 2. Проблема

Малый и средний сервисный бизнес теряет деньги из-за:

- пропущенных звонков
- отсутствия ответа вне рабочего времени
- медленной обработки типовых вопросов
- ручной записи в календарь/CRM
- отсутствия прозрачной аналитики по звонкам

Существующие voice AI платформы часто:

- слишком технические для SMB
- плохо объясняют, почему звонок или сценарий сломался
- становятся дорогими на объёме
- слабо решают handoff на оператора

## 3. Почему сейчас

- voice AI API и telephony API созрели достаточно для коммерческого продукта
- open-core модель даёт доверие, adoption и lower CAC через разработчиков/агентства
- рынок SMB всё ещё слабо закрыт именно по outcome-driven голосовой автоматизации
- агентствам и интеграторам нужен white-label слой, а не только affiliate-ссылка

## 4. Целевые пользователи

### Primary ICP

1. клиники, стоматологии, бьюти и wellness
2. рестораны и заведения с бронью
3. home services

### Secondary ICP

- агентства и web-студии, продающие автоматизацию клиентам
- разработчики, которым нужен self-hosted/open-core voice runtime

### Ключевые роли

- владелец бизнеса
- оператор/администратор
- агентство-реселлер
- технический интегратор

## 5. Ценность продукта

### Для SMB

- меньше пропущенных обращений
- больше подтверждённых записей
- ответы 24/7
- меньше ручной рутины
- понятная окупаемость

### Для агентств

- перепродажа под своим брендом
- много клиентских аккаунтов в одной панели
- шаблоны под вертикали
- прозрачный usage и margin control

### Для разработчиков

- self-hosted ядро
- API-first платформа
- provider abstraction
- возможность строить свои вертикальные продукты поверх ядра

## 6. Продуктовая стратегия

Продукт разделяется на 3 издания:

### OSS Core

- self-hosted execution engine
- API
- базовые workflow
- базовые провайдер-адаптеры
- базовые логи/транскрипты

### Cloud / Pro

- hosted control plane
- аналитика, replay, QA
- биллинг
- hosted integrations
- командные функции
- расширенная наблюдаемость

### Agency

- white-label
- subaccounts
- partner dashboard
- usage по клиентам
- margin / rebilling инструменты

## 7. MVP-фокус

Первый продаваемый сценарий:

`AI receptionist + booking + human transfer`

Что входит в MVP:

- приём звонка
- определение интента
- ответы на FAQ
- запись в календарь
- сбор имени/телефона/слота
- перевод на оператора
- транскрипт и итоги звонка
- webhook/API события
- базовая аналитика по звонкам и конверсии

Что не входит в MVP:

- полноценный visual flow builder enterprise-уровня
- мобильное приложение
- сложный multi-region enterprise deployment
- SSO/SAML
- full white-label studio для всех ролей
- большое количество вертикалей одновременно

## 8. Метрики успеха

### Product metrics

- время до первого рабочего агента: <= 15 минут
- время публикации изменения агента: <= 2 минут
- средняя длительность настройки SMB-сценария: <= 30 минут

### Business metrics

- trial -> paid conversion: >= 15% на раннем этапе
- активация: >= 60% trial-аккаунтов завершают первую публикацию агента
- churn: <= 5% в месяц на первых когортах
- доля звонков, обработанных без человека: >= 40% для типового FAQ/booking сценария

### Quality metrics

- median time-to-first-audio: <= 1200 ms
- error rate по критичным call flows: < 3%
- доля звонков с корректным summary/event export: >= 98%

## 9. Каналы роста

- direct sales в выбранную вертикаль
- design partners
- контент и кейсы
- affiliate program
- agency/reseller channel
- OSS adoption -> paid upgrade

## 10. Ключевые риски

- telephony и voice cost могут съесть маржу
- сложность UX для нетехнических пользователей
- неудачный выбор первичного ICP
- слишком широкий MVP
- зависимость от внешних voice/LLM provider'ов

## 11. Ключевой принцип

`VoiceAgent` должен продаваться как outcome platform, а не как очередной AI builder.
